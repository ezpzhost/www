Getting started

Everything on ezpz.host happens through an HTTP API. Here's the whole flow, start to finish.

1. Generate a token

There's no signup. Generate 64 hex characters locally. That string is your token and your only credential:

openssl rand -hex 32

ezpz.host services only ever store a SHA-256 hash of your token, so it cannot be recovered if lost. Keep it safe.

2. Authenticate

Send it as a bearer token on every request:

Authorization: Bearer <token>

3. Fund a size

Pick a size (see Servers) and how many days you want:

curl -X POST https://api.ezpz.host/billing/fund \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"size":"xs","days":30}'

This returns a fresh bitcoin address and the exact amount to send, at a rate locked for 24 hours.

4. Pay, then watch it credit

curl -H "Authorization: Bearer <token>" \
  https://api.ezpz.host/billing/status

Once your payment confirms, the days show up here.

5. Launch a server

curl -X POST https://api.ezpz.host/vps/create \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "region": "usa",
    "size": "xs",
    "image": "ubuntu",
    "ssh_key": "ssh-ed25519 AAAA..."
  }'

Creating a server requires at least one funded day of that size, otherwise this returns 402. Fund it first.

6. Wait for it to come online

curl -H "Authorization: Bearer <token>" \
  "https://api.ezpz.host/vps/status?id=<id>"

Poll until status is active, then:

ssh root@<ipv4>
