Token

Generate 64 hex characters locally, that string is your token and your only credential. There is no signup, no email, and no account to create. ezpz.host services only ever store a SHA-256 hash of your token, so it cannot be recovered if lost. Keep it safe.

openssl rand -hex 32

There's no password reset, no "forgot your token" flow, and no support ticket that can get it back for you, the hash we store can't be reversed. Back it up somewhere durable before you fund anything.

Using it

  1. Send authenticated requests with the header: Authorization: Bearer <token>
  2. POST /billing/fund with a size and number of days to get a bitcoin address and a locked quote.
  3. Pay the address; GET /billing/status to watch the days credit.
  4. POST /vps/create to launch a server of a size you've funded days for.
